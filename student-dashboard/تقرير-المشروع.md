## 1. نظرة عامة

**نظام شامل** هو منصة متكاملة لأتمتة وإدارة بيانات حلقات تحفيظ القرآن الكريم،
مبنية فوق منصة **إنجازي** (`injaazy.com`). النظام يتبع لـ **جمعية الوحيين**
(Corporation ID: 118 — الفرع الرجالي: 171).

يحقّق النظام ثلاثة أهداف رئيسية:

1. **الأتمتة** — مزامنة دورية للبيانات (حلقات، طلاب، منشآت، معلمين) مع جداول Google Sheets.
2. **الإدارة** — لوحة تحكم إدارية لإدارة المهام المجدولة واستيراد الطلاب ومراجعة السجلات.
3. **العرض للطالب** — لوحة "شامل" تعرض للطالب تقدّمه في الحفظ والحضور عبر رابط خاص به.

## 3. المكوّنات الرئيسية

### 3.1 مكتبة `enjazi/` — العميل الأساسي

مكتبة Python مسؤولة عن كل التواصل مع API إنجازي (`api.injaazy.com/front_app_api/v1`).

| الملف | الوظيفة |
|---|---|
| `client.py` | عميل HTTP يحاكي بصمة Chrome TLS عبر `curl_cffi` لتجاوز حماية Cloudflare، ويرفق ترويسات السياق (`x-behalf-*`) ويطبّق تحديد معدّل الطلبات. |
| `auth.py` | تسجيل الدخول وإدارة الـ Bearer Token مع تخزين مؤقت في `.token_cache.json` (صلاحية افتراضية 720 ساعة). |
| `exceptions.py` | أصناف الأخطاء (`AuthenticationError`, `NetworkError`, ...) ومُحوّل استجابات HTTP. |
| `api/` | وحدات الـ API: `base`, `centers`, `episodes`, `institutions`, `reports`, `students`, `teachers`. |
| `utils/` | `logger.py` (loguru) و `rate_limiter.py` (تأخير افتراضي 5 ثوانٍ بين الطلبات). |

**نقاط بارزة:**
- يستخدم `curl_cffi` بانتحال `chrome124` لتمرير فحص البوتات.
- التحكم في نطاق العمليات يتم عبر ترويسات `x-behalf-id` / `x-behalf-on` / `x-current-role`.

### 3.2 الخادم `backend/` — FastAPI

خادم الـ API المركزي. نقطة الدخول `backend/main.py` تشغّل التطبيق وتُهيّئ
قاعدة البيانات وتبدأ المجدوِل عند الإقلاع (عبر `lifespan`).

**المُوجِّهات (Routers):**

| المسار | الوظيفة |
|---|---|
| `/api/jobs` | إدارة المهام المجدولة (إنشاء، تعديل، حذف، تشغيل يدوي). |
| `/api/data` | جلب وتحديث البيانات المخزّنة مؤقتاً (الفروع، المنشآت، الحلقات، الطلاب، المعلمين). |
| `/api/logs` | استعراض سجلّات تنفيذ المهام وتفاصيل الطلاب المستوردين. |
| `/api/import` | استيراد الطلاب من CSV أو JSON، مع دعم بثّ النتائج الحيّة عبر SSE. |
| `/api/student-page/{sub_url}` | تجميع بيانات لوحة الطالب (يُغذّي `student-dashboard`). |

**`student_page` endpoint** هو الأهم للوحة الطالب — يقوم بـ:
1. حلّ `sub_url` إلى بيانات الطالب من Google Sheets.
2. جلب تقرير الطالب الأسبوعي + الشهري من إنجازي.
3. حساب تقدّم حفظ القرآن من أرقام الصفحات (`pages_numbers`).
4. حساب معدّل الحفظ اليومي = (الصفحات الشهرية ÷ أيام الحضور).
5. إرجاع JSON موحّد يجمع البيانات الشخصية + التقرير + التقدّم.

### 3.3 لوحة الإدارة `frontend/`

تطبيق Next.js (إصدار 16.2.3) لإدارة النظام، يستخدم `shadcn/ui` و `axios`
و `lucide-react` و Tailwind CSS v4.

**الصفحات:** الرئيسية، الحلقات، المنشآت، الاستيراد، المهام، السجلات (+ تفاصيل سجل)، الطلاب، المعلمين.

### 3.4 لوحة الطالب `student-dashboard/` — "شامل"

تطبيق Next.js (إصدار 16.2.6) — وهو **المجلد الحالي** الذي يُكتب فيه هذا التقرير.
يعرض لكل طالب صفحة تقدّم خاصة عبر مسار ديناميكي `/[subUrl]`.

**البطاقات (Components):**

| المكوّن | المحتوى |
|---|---|
| `HeaderCard` | شعار الجمعية + معلومات الطالب (الاسم، الحلقة، المنشأة، المعلم، الخطة). |
| `DailyPulseCard` | "نبضة اليوم" — درس الحفظ والمراجعة لليوم الحالي. |
| `WeeklyCard` | مخطّط أعمدة للحفظ الشهري + ملخّص الحضور (حاضر/غائب/معذور). |
| `BigPictureCard` | "الصورة الكبرى" — نسبة تقدّم المصحف + متنبّئ سرعة الختم + حاسبة ختم تفاعلية. |
| `HeatmapCard` | خريطة حرارية لحالة حفظ الصفحات (راسخ/جيد/يحتاج مراجعة/لم يُحفظ). |
| `ShareCard` | "ملخّص التقدّم" — بطاقة قابلة للمشاركة كصورة PNG مع اسم الشهر. |
| `ui/Badge`, `ui/ProgressBar` | عناصر واجهة مشتركة. |

**ملاحظات تقنية:**
- واجهة بالعربية بالكامل (RTL)، خطّا `Cairo` و `Amiri` من Google Fonts.
- التصميم بـ "تنسيقات سطرية" (inline styles) وألوان `oklch`، بطابع شهادة تقليدية.
- `ShareCard` يستخدم مكتبة `html-to-image` لتصدير البطاقة صورةً
  (مشاركة عبر Web Share API على الجوال، أو تنزيل مباشر على سطح المكتب).
- مصدر بيانات اللوحة: `lib/api.ts` يستدعي `NEXT_PUBLIC_API_URL`، والأنواع في `lib/types.ts`.

## 4. قاعدة البيانات والنماذج

قاعدة بيانات **SQLite** (`enjazi.db`) عبر **SQLAlchemy 2.0**. الجداول تُنشأ تلقائياً عند الإقلاع.

| النموذج | الجدول | الوصف |
|---|---|---|
| `Job` | `jobs` | مهمة مجدولة (الاسم، النوع، تعبير cron، المعاملات، التفعيل). |
| `JobLog` | — | سجلّ تنفيذ مهمة (الحالة، المُطلِق، وقت البدء/الانتهاء، النتيجة). |
| `DataCache` | — | تخزين مؤقت لبيانات الفروع/المنشآت/الحلقات/الطلاب/المعلمين. |
| `StudentImport` | — | سجلّ كل طالب تمّ استيراده ضمن مهمة. |
| `cache` | — | نموذج تخزين مؤقت إضافي. |

> ملاحظة: `sheets_lookup` يستخدم أيضاً تخزيناً مؤقتاً في الذاكرة بصلاحية 5 دقائق.

## 5. المهام المجدولة (Scheduler)

يعتمد النظام **APScheduler** (`AsyncIOScheduler`، توقيت `Asia/Riyadh`).
المهام تُحمّل من قاعدة البيانات عند الإقلاع وتُجدوَل بتعابير cron.

**أنواع المهام المتاحة (7 أنواع):**

| النوع | الوظيفة |
|---|---|
| `sync_episodes` | مزامنة الحلقات. |
| `export_students` | تصدير الطلاب. |
| `import_from_sheets` | استيراد طلاب من Google Sheets. |
| `export_facilities_to_sheets` | تصدير المنشآت إلى Sheets. |
| `export_episodes_to_sheets` | تصدير الحلقات إلى Sheets. |
| `export_students_to_sheets` | تصدير الطلاب إلى Sheets. |
| `log_daily_indicators` | تسجيل المؤشرات اليومية. |

كل تنفيذ يُسجَّل في `JobLog` بحالته (`running` → `success`/`failed`)،
ويمكن تشغيل أي مهمة يدوياً عبر `POST /api/jobs/{id}/run`.

## 6. تدفّق البيانات — رحلة طلب لوحة الطالب

```
الطالب يفتح  /{subUrl}
   │
   ▼
student-dashboard → fetchStudentPage()  → GET /api/student-page/{subUrl}
   │
   ▼
backend: resolve_sub_url()  → Google Sheets ("ربط الطلاب" + "بيانات الطلاب")
   │   يُرجِع: enjazi_student_id, episode_id, institution_id, البيانات الشخصية
   ▼
backend: EnjaziClient + get_valid_token()
   │   → تقرير أسبوعي + تقرير شهري + دروس اليوم  من api.injaazy.com
   ▼
backend: حساب تقدّم القرآن + معدّل الحفظ اليومي
   │
   ▼
JSON موحّد  → عرضه في البطاقات الستّ
```

## 8. النشر (Deployment)

- **الخادم**: حاوية Docker مبنية من `python:3.12-slim` (`Dockerfile`)، تشغّل
  Uvicorn على المنفذ 8000، وقاعدة البيانات في حجم `db_data` مثبّت على `/app/db`.
- **التنسيق**: `docker-compose.yml` يشغّل خدمتي `backend` و `frontend` على شبكة
  `dokploy-network` (منصّة Dokploy).
- **عناوين الإنتاج**: الواجهة تتصل بـ `https://backendinjzi.alwahyain.org`.
- **الإعدادات الحسّاسة**: في ملف `.env` (مُستبعَد من Git) — بيانات الدخول لإنجازي،
  ومفاتيح حساب خدمة Google، ومعرّف الجدول.

## 9. ملاحظات ونقاط للانتباه

1. **تعدّد الواجهات**: يوجد مجلدان للواجهة — `frontend/` (إدارة) و
   `student-dashboard/` (طالب). يبدو `frontend/` يحوي نسخة سابقة من
   لوحة الطالب أيضاً؛ يُستحسن توحيد البنية أو توثيق الفصل بوضوح.
2. **اتفاقية المشروع**: ملف `AGENTS.md` ينبّه أن إصدار Next.js المستخدم قد يختلف
   عن المتوقّع، ويطلب قراءة التوثيق داخل `node_modules/next/dist/docs/` قبل الكتابة.
3. **الاعتماد على Google Sheets** كمصدر لربط الطلاب يجعل توفّر النظام مرتبطاً
   بإتاحة الجدول وصحّة ترتيب أعمدته (الأعمدة مُفهرَسة بمواقع ثابتة: M، N، إلخ).
4. **اعتبارات أمنية**: `CORS` مضبوط افتراضياً على `*`؛ يُفضّل تقييده عبر
   `ALLOWED_ORIGINS` في بيئة الإنتاج.
5. **ملفات نصّية متفرّقة** في الجذر (`p.txt`، `injaz.txt`، `re.txt`، ...) تبدو
   مسوّدات/استكشافات — يُستحسن أرشفتها أو حذفها لتنظيف المستودع.
