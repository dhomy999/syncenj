## 1. المصادقة والترويسات

### تسجيل الدخول
**`POST auth/login`**
```json
{ "username": "2366680136", "password": "********", "country_id": 1 }
```
- `country_id` **إلزامي** (بدونه `422: حقل country id مطلوب`).
- الاستجابة تُعيد `data.access_token`.

### الترويسات العامة
```http
Authorization: Bearer {access_token}
Content-Type: application/json
Accept: application/json
```

### الترويسات الخاصة بالسياق
| الترويسة | متى تُستخدم | ملاحظة |
|----------|------------|--------|
| `x-episode-id: {id}` | كل نقاط الحلقة/الطالب | إلزامية — بدونها `422: x-episode-id header is required` |
| `x-student-id: {id}` | نقاط الطالب المفردة | تُمرَّر مع الحلقة |

> معرّف الطالب يُمرَّر أيضاً في **مسار** نقاط الطالب: `teacher/students/{student_id}/...`

### تجديد التوكن / انتهاء الجلسة
- `POST auth/refresh` — لتجديد التوكن عند انتهائه (401).

## 2. مرجع نقاط النهاية الكامل

### 2.1 المصادقة (`auth/`)
| Method | Endpoint | الوظيفة |
|--------|----------|---------|
| POST | `auth/login` | تسجيل الدخول |
| POST | `auth/logout` | تسجيل الخروج (إبطال التوكن) |
| POST | `auth/refresh` | تجديد التوكن |
| POST | `auth/check_username` | التحقق من اسم المستخدم |

### 2.2 الحلقات (`teacher/`)
| Method | Endpoint | الترويسات | الوظيفة |
|--------|----------|-----------|---------|
| GET | `teacher/episodes-listing/active` | — | الحلقات النشطة |
| GET | `teacher/episodes-listing/all` | — | كل الحلقات |
| GET | `teacher/episode/students` | `x-episode-id` | كشف طلاب الحلقة + حالة التحضير |
| GET | `teacher/episode/form-data/` | `x-episode-id` | بيانات نموذج الحلقة |
| GET | `teacher/episode/from-verse-permissions` | `x-episode-id` | صلاحيات تعديل بداية كل ركن |
| **POST** | **`teacher/episode/attendece`** | `x-episode-id` | **إرسال/اعتماد التحضير** |

> ⚠️ ملاحظة إملائية: مسار الإرسال هو `attendece` (بدون p). `attendecep` الموجود في التوثيق القديم **لا يقبل POST** (405).

### 2.3 الطالب والدروس (`teacher/students/{id}/`)
| Method | Endpoint | الحمولة/المطلوب | الوظيفة |
|--------|----------|------------------|---------|
| GET | `teacher/students/{id}/lessons` | `x-episode-id` | درس اليوم لكل الأركان + الحالة |
| GET | `teacher/students/{id}/history-lessons?date_of=YYYY-MM-DD` | `x-episode-id` | **السجل الفعلي المسجَّل** ليوم معيّن |
| GET | `teacher/students/{id}/plan` | `x-episode-id` | خطة الطالب والأركان |
| **POST** | **`teacher/students/{id}/recite`** | `{pillar_id}` | **تسجيل "تم التسميع"** |
| **POST** | **`teacher/students/{id}/change-starting`** | `{pillar_id, from_verse_id}` | **تغيير نقطة بداية ركن** |
| **POST** | **`teacher/students/{id}/extra-recite`** | `{pillar_id, from_verse_id, to_verse_id}` | **إضافة مقطع (غير متزامن)** |
| POST | `teacher/students/{id}/update-next-lesson` | `{lessons:[...]}` | تعديل الدرس القادم |
| POST | `teacher/students/{id}/cancel-recite` | (للتراجع) | إلغاء التسميع |
| POST | `teacher/students/{id}/extra-recite/cancel` | (للتراجع) | إلغاء مقطع إضافي |

### 2.4 الثوابت
| Method | Endpoint | الوظيفة |
|--------|----------|---------|
| GET | `teacher/students/plan-management/constants` | الأركان، الاتجاهات، طرق الدراسة، `pillar_ids` |
| GET | `general/fetch-sour` | قائمة السور |

## 3. طريقة التحضير

### الخطوات
```
1. login  → token
2. GET teacher/episodes-listing/active      → اختيار episode_id
3. GET teacher/episode/students (x-episode-id) → student_id + الحالة
4. POST teacher/episode/attendece (x-episode-id) → الإرسال
```

### الحمولة
**`POST teacher/episode/attendece`** + `x-episode-id: {episode_id}`
```json
{
  "students": [
    { "student_id": 191954, "attend_type": "attend" }
  ]
}
```

### قواعد التحقق
| الحقل | القاعدة |
|-------|---------|
| `students` | مصفوفة إلزامية |
| `students[].student_id` | إلزامي (= `items[].id` من كشف الطلاب) |
| `students[].attend_type` | إلزامي — القيم: `attend` / `late` / `excused` / `absent` |

### ⚠️ نطاق الانفجار (Blast Radius) — الأهم
الاستدعاء **يعتمد الجلسة كاملة، وليس تحديثاً إضافياً**:
> إرسال طالب واحد فقط كـ `attend` يجعل **كل بقية طلاب الحلقة `absent` تلقائياً**.

**لذلك في الأتمتة: اجمع قائمة الحضور الكاملة لكل الحلقة وأرسلها دفعة واحدة**، وإلا ستُغيّب طلاباً حقيقيين خطأً.

### استجابة النجاح
```json
{ "success": true, "status": 200, "message": "تم تسجيل حالة التحضير بنجاح" }
```
بعدها يصبح `attended_today = true` و`has_lessons = true` (شرط لتسجيل التسميع).

## 4. طريقة إدخال التسميع

> **الشرط المسبق:** يجب أن يكون الطالب محضَّراً اليوم أولاً (التحضير يفعّل `has_lessons`).

### 4.1 الحالة أ — تسجيل الركن بنطاقه الافتراضي
لتسجيل ما جدوله التطبيق تلقائياً (بلا تغيير نطاق):

**`POST teacher/students/{id}/recite`** + `x-episode-id`
```json
{ "pillar_id": 2 }
```
- يستخدم نطاق درس اليوم (`from`/`to`) كما هو.
- النتيجة: `done=true`, `current_state="RECITE"`, ويُحتسب التقييم تلقائياً.

### 4.2 الحالة ب — تسجيل بنطاق مخصّص (بداية مختلفة)
لتغيير نقطة البداية ثم التسجيل:

**الخطوة 1 — ضبط البداية:** `POST teacher/students/{id}/change-starting`
```json
{ "pillar_id": 3, "from_verse_id": 5105 }
```
- ينقل بداية الركن إلى الآية المطلوبة، ويُعيد النظام حساب النهاية تلقائياً حسب طريقة الدراسة (قد لا تطابق النهاية التي تريدها).

**الخطوة 2 — التسجيل:** `POST teacher/students/{id}/recite`
```json
{ "pillar_id": 3 }
```

> ⚠️ **`recite_to_verse_id` في حمولة `recite` يُتجاهَل** — لا يمكن تحديد النهاية من خلاله.

### 4.3 الحالة ج — تمديد النطاق لنهاية أبعد (Extra Recite)
إذا كانت النهاية المحسوبة أقصر من المطلوب، أضِف مقطعاً من (نهاية التسجيل + 1) إلى النهاية المطلوبة:

**`POST teacher/students/{id}/extra-recite`** + `x-episode-id`
```json
{ "pillar_id": 3, "from_verse_id": 5178, "to_verse_id": 5241 }
```
- **غير متزامن:** الاستجابة تحوي `"queued": true`. لا يظهر فوراً في `.../lessons`.
- **تحقّق منه عبر:** `GET teacher/students/{id}/history-lessons?date_of=YYYY-MM-DD` (يظهر كقيد مستقل ضمن نفس الركن).

### 4.4 التحقق من النتيجة
**`GET teacher/students/{id}/history-lessons?date_of=YYYY-MM-DD`** — يُظهر السجل الفعلي المسجَّل، مع `from`/`to` (surah_id, verse_order, verse_id) و`pages` و`rating` لكل قيد.

### مثال متكامل (ما نُفِّذ فعلياً)
| الركن | الطريقة | النطاق المسجَّل | verse_id |
|-------|---------|----------------|----------|
| حفظ (2) | 4.1 recite | القصص 71 → 77 | 3323→3329 |
| مراجعة (3) | 4.2 change-starting + recite | المجادلة 1 → الصف 14 | 5105→5177 |
| مراجعة (3) | 4.3 extra-recite | الجمعة 1 → التحريم 12 | 5178→5241 |

النتيجة: المراجعة المجمّعة = المجادلة 1 → التحريم 12 (متصلة).

## 5. الأركان (Pillars) والاتجاهات

### الأركان (من `plan-management/constants`)
| id | code | الاسم | الوصف |
|----|------|-------|-------|
| 1 | TE | تلاوة | تصحيح التلاوة وتحسين الأداء |
| 2 | SAV | حفظ | الحفظ الجديد (= "التسميع") |
| 3 | REV | مراجعة | المراجعة |
| 4 | SM-REV | تثبيت | التثبيت |
| 5 | CUM-CONS | تثبيت تراكمي | — |
| 6 | NEXT-TE | تلاوة الدرس القادم | — |
| 7 | ADD-REV | مراجعة إضافية | — |
| 8 | CUM-REV | مراجعة تراكمية | — |

`pillar_ids` map: `{"TE":1,"SAV":2,"REV":3,"SM-REV":4,"CUM-CONS":5,"NEXT-TE":6,"ADD-REV":7,"CUM-REV":8}`

### الاتجاهات (directions)
| id | الاسم | الوصف |
|----|-------|-------|
| 1 | من الفاتحة | من الفاتحة إلى الناس (تصاعدي) |
| 2 | من الناس | من الناس إلى الفاتحة (تنازلي) |

### ⚠️ صلاحيات تعديل البداية (`from-verse-permissions`)
ليست كل الأركان قابلة لتغيير البداية. **اقرأ `teacher/episode/from-verse-permissions` أولاً**:
| الركن | `can_edit_from_verse` |
|-------|------------------------|
| حفظ (2) | ❌ **false** — مقفل على الخطة، لا يمكن نقله |
| مراجعة (3) | ✅ true |
| تثبيت (4) | ✅ true |

> **ملاحظات جدولة:** ليس كل ركن مجدولاً كل يوم (قد لا يظهر التثبيت في `lessons`). درس اليوم في `lessons[]` قد يحوي أركاناً أقل من أركان الخطة.

## 6. مرجعية verse_id

`verse_id` = **الترقيم العالمي المتسلسل للآيات** (1–6236، مصحف حفص، البسملة لا تُعدّ آية إلا في الفاتحة).

**الصيغة:** `verse_id = (مجموع آيات السور قبل السورة) + رقم الآية`

**نقاط تحقّق مؤكَّدة عملياً:**
| الموضع | verse_id |
|--------|----------|
| القصص 28:71 | 3323 |
| القصص 28:77 | 3329 |
| العنكبوت 29:69 | 3409 |
| المجادلة 58:1 | 5105 |
| الجمعة 62:1 | 5178 |
| التحريم 66:12 | 5241 |
| الكهف 18:1 | 2141 |
| الكهف 18:50 | 2190 |
| الكهف 18:59 | 2199 |

> ⚠️ **خطأ بآية واحدة يفسد السجل بصمت.** اشتقّ `verse_id` من مصدر موثوق (جدول عدّ الآيات المطابق، أو `general/fetch-sour`) وتحقّق بنقاط مرجعية معروفة قبل الاعتماد.

## 7. رموز الاستجابة

| الرمز | المعنى في هذا السياق |
|-------|----------------------|
| 200 | نجاح (تحقّق من `success` و`message`؛ وقد يحوي `"queued": true` للعمليات غير المتزامنة) |
| 422 | حقل مطلوب مفقود/غير صالح — التفاصيل في `errors[].param` |
| 405 | Method غير مسموح (مسار/طريقة خاطئة) |
| 502 / 400 | مسار غير مطابق (fallback) |
| — | استجابة `{"message":"app is running","service":"laravel-api"}` = مسار GET غير مطابق (fallback صحّة) |

**نموذج الخطأ:**
```json
{ "success": false, "status": 422, "message": "...", "errors": [ { "param": "students", "message": "..." } ] }
```

## 8. مسارات التراجع (Undo)

| العملية | نقطة التراجع |
|---------|--------------|
| إلغاء تسميع ركن | `cancel-recite` / إجراء `MARK_NOT_RECITED` |
| إلغاء مقطع إضافي | `extra-recite/cancel` |
| إرجاع بداية ركن | `change-starting` بالقيمة **الأصلية** (احفظها قبل التغيير!) |
| إلغاء الحضور | إعادة إرسال `attendece` بالحالة الصحيحة لكل الطلاب |

> **مهم:** `change-starting` يعدّل الخطة بشكل دائم. لإرجاع الحالة يجب تخزين `from_verse_id` الأصلي قبل التغيير ثم إعادته (مثال: المراجعة الأصلية كانت تبدأ من القصص 76 = `from_verse_id 3328`).

## 9. قائمة تحقق الأتمتة

- [ ] **التوكن:** تعامل مع الانتهاء عبر `auth/refresh` وأعِد المحاولة عند 401.
- [ ] **الاستهداف:** تحقّق من `x-episode-id` و`student_id` مقابل قيمة متوقعة قبل كل كتابة (الحساب قد يكون إشرافياً يرى عدة معلّمين).
- [ ] **التحضير الكامل:** أرسل قائمة حضور الحلقة **كاملة** (وإلا يُغيَّب الباقي تلقائياً).
- [ ] **الترتيب:** التحضير → التسميع الأساسي → المقطع الإضافي.
- [ ] **الصلاحيات:** اقرأ `from-verse-permissions`؛ لا تحاول تغيير بداية ركن مقفل (حفظ).
- [ ] **الخطة:** قرّر — هل تُرجع `change-starting` للأصل (نطاق مؤقت)، أم تُبقيه (تحريك الخطة)؟ خزّن القيمة الأصلية دائماً.
- [ ] **النهاية:** لا تعتمد على `recite_to_verse_id` (يُتجاهَل)؛ استخدم `extra-recite` لتحديد نهاية أبعد.
- [ ] **اللاتزامن:** بعد `extra-recite` تحقّق عبر `history-lessons` (لا `lessons`).
- [ ] **الـ idempotency:** لا يوجد مفتاح idempotency ولا dry-run — **اقرأ الحالة وتحقّق قبل أي إعادة محاولة** لتفادي التكرار.
- [ ] **أيام العمل:** احترم `is_work_today` و`selected_days` في الخطة.
- [ ] **verse_id:** اشتقّه من مصدر موثوق وتحقّق بنقاط مرجعية.
- [ ] **التدقيق:** سجّل كل كتابة (قبل/بعد) لتمكين التراجع الآلي.
